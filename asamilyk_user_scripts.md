**[ Сценарий 1 - Ознакомление с проектом ]**

1.	Экран приветствия игрока
2.	Возможность настройки начальных опций (звук,музыка)
3.	Первичное ознакомление пользователя с игрой 
4.	Обучение и ознакомление с игровым процессом 
5.	Проделывание пользователем первых действий в игровом процессе под наблюдением (контроль понятности интерфейса и игрового процесса)
6.  Сохранение прогресса 
7.   Самостоятельное погружение в игровой процесс пользователем с дальнейшими выходами из игры (при надобности) 

**[ Сценарий 2 - Работа с примерами]**
1. Посещение пользователем определенной локации в игре для осуществления главного игрового процесса (решение арифметических выражений)
2.	Выведение арифметического выражения пользователю
3.	Пользователь вводит ответ
4.	Если пользователь ввел неправильный ответ, даётся еще 1 возможность ввести правильный (при повышении сложности такой возможности уже не будет)
5.	Если ответ корректен, пользователь получает вознаграждение в виде игровой валюты 
6.	Пользователь после достижения определенного количества игровой валюты может повышать уровень сложности 
7.  Также возможно добавление новых арифметических знаков в выражениях для увеличения сложности  

**[ Сценарий 3 - Работа с игровой локацией]**
1.	Пользователю доступна игровая локация 
2.	При нажатии на здание пользователем будет высвечиваться характеристика здания и возможность совершать какие-либо сделки или действия. Также улучшать здание для повышения уровня сложности
3.	Возможность видеть игровую валюту 
4.	Возможность зайти в меню
5.  Возможность открыть «Справочную» для нахождения нужной или актуальной информации 
6.  Возможность расширения игровой локации для большего количества действий в игре

**[ Сценарий 4 – Меню]**
1.	Предусматривается отдельная вкладка «Статистика» для пользователя
2.	Будет предусмотрено так же опция «Достижения» для большей мотивации пользователя играть и получать награды. Отдельная статистика:
a)	Время, проведенное в игре
b)	Сколько игровой валюты получено
c)	Сколько игровой валюты потрачено
d)	Данный уровень сложности 
e)	Сколько ошибок было совершено в арифметических примерах
3.	Опция «Настройки», подразумевающие включения/выключения звука, музыки (в будущем будет добавлена опция смена языка)
4.	Опция «Персонаж». Подразумевается, что пользователь создаст своего персонажа (внешний вид, одежда). Некоторые элементы одежды будут влиять на игровой процесс (к примеру шапка «мешочек с золотом» будет давать возможность зарабатывать больше денег) 


**[ Сценарий 5 – Валюта]**
1.	Пользователю доступна игровая валюта в виде золота, которая добывается при решение арифметических выражений
2.	Пользователю доступна игровая валюта в виде изумрудов, которая является более редкая и может быть получена только в ходе выполнения достижений или особых арифметических выражений
3.	Золото может быть потрачена на улучшение построек, открытии территории и других возможностей (при достижении определенного уровня), позволяющие усовершенствовать игровой процесс
4.	Изумруды могут быть потрачены на более «изысканные» улучшения, такие как, изменение внешнего вида построек или ускорение процесса, при котором можно улучшать здания (не достигая определенного уровня)
5.	Возможность обменивать изумруды на золото при должной необходимости пользователя 
6.	При дальнейшем развитии своего города, пользователь будет иметь возможность воздвигать постройки, которые в свою очередь будут приносить ему валюту (золото/изумруды)
7.	При достижении высокого уровня, пользователю придется платить за выполнение арифметических выражений, тем самым при успешном выполнении игрок будет выходить в плюс, а если нет – то в минус. То есть появляется большая мотивация опытного игрока не ошибаться, ведь «на кону» стоят его же золото. 
8.	*в будущем будет добавлена система «донат», при котором пользователь сможет за реальные деньги приобретать игровую валюту*


**[ Сценарий 6 – Уровень]**
1.	Пользователю доступен игровой уровень 
2.	Игровой уровень формируется при достижении определенного количества «ОП» в шкале «Уровень». Эти «ОПы» можно получить при улучшении города и выполнения заданий
3.	С развитием города, уровень игрока тоже растет, давая больше возможностей в игре
4.	Игровой уровень связан со сложностью арифметических выражений, т. е. чем больше уровень, тем больше улучшений зданий игрок может совершить, тем сложнее становится игровой процесс 
5.	Некоторые постройки и возможности могут быть использованы только при достижения определенного уровня 
6.	Возможен обмен золота на валюту «ОП»
7.	При каждом достижении нового уровня, количество «ОП», которое нужно получить для повышения уровня, возрастает по алгебраической прогрессии 


**[ Сценарий 7 – Магазин]**
1.	Пользователю доступна вкладка «Магазин»
2.	В магазине пользователь имеет возможность:
a)	Купить улучшение 
b)	Открыть постройку
c)	Купить одежду (для персонажа)
d)	Купить декорацию 
(все эти аспекты прямо или косвенно влияют на мотивацию пользователя играть и обучаться одновременно)
3.	Размещение объектов из магазина будет предусмотрено самим разработчиком 
4.	Некоторые вкладки в магазине могут быть доступны только после достижения определенного уровня или пользователь должен потратить изумруды на открытие этой вкладки 
5.	В магазине доступен курс, по которому игрок может обменять изумруды на золото, а золото на «ОП»
